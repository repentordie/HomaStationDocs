# Основы
## Терминология
**Mapping (Маппинг)** - Процесс создания карт.

**Map (Мапа, Карта)** - Собственно, карта. В одном раунде(смене) присутствует намного больше, чем одна карта. Станция и всё что её окружает - это одна **Карта**, ЦК - другая **карта**. Игра переносит игроков между ними когда требуется, но помимо этого у **Мап** нет никакого интерфейса чтобы взаимодействовать *друг с другом*.

**Grid (Грид)** - это субъединица **Карты**. Если мы говорим о том, что игровой мир - **Карта**, то Шаттл Карго, Эвак, Дереликты и, непосредственно, сама станция - это **Грид**.
Каждый **Грид**  живёт отдельно от себя и может создаваться динамически  - распилите станцию пополам, и она превратится в два **Грида**. *Важное уточнение* - это работает в одну сторону. Когда ты пристыковываешь шаттл к станции, они не становятся одним **Гридом**, они просто двигаются вместе.

*Гриды НЕ МОГУТ ПУТЕШЕСТВОВАТЬ МЕЖДУ МАПАМИ, ЭТО ХИТРЫЕ КОДИНГ ТРЮКИ.*

## Консольные команды и Инструменты
### Подготовка к созданию карт
- `mapping < ID > < ПУТЬ >` - Запускает процесс редактора карты находящейся по ПУТИ из директории сервера.
	- Чтобы создать новую карту достаточно ввести только ID. ID должен быть свободным, т.е. не использоваться другой картой на сервере. Чтобы избежать конфликтов используйте достаточно высокое значение.
	-  Чтобы изменить карту, которую вы до этого сохраняли вы можете вместо полного пути, использовать название файла вашей карты. 
- `mapinit < ID >` - Размораживает карту для тестирования (по умолчанию, все карты созданные через mapping заморожены). Перед использованием, **сохраните карту на сервер**!
- `savemap < ID > < НАЗВАНИЕ >` - Сохраняет локальную копию указанной карты.
	- Сохраненные карты могут быть найдены в папке `bin/Content.Server/data` и находятся отдельно от тех карт, которые используются репозиторием в папке  `Resources/Maps`
	*Два важных момента:*
	1. Карту можно сохранять...<h1>**ТОЛЬКО В ЗАМОРОЖЕННОМ СОСТОЯНИИ**</h1>
	Если вы попытаетесь сначала протестировать и разморозить карту, а потом сохранить её - игра просто откажется это делать. 

	2. На данный момент, пока сервер в тестовой конфигурации, у вас есть **РЕАЛЬНЫЙ РИСК** потерять свою работу, если вовремя не сказать Лэпу перенести карту в обычные файлы. Сервер затирает всё, что не находится на ГитХабе, а новые карты регистрируются в консоли только после перезапуска, что сотрёт их.

### Работа с картами
- `lsmap` - Выдаёт список карт в раунде (смене)
- `addmap < ID >`  - Добавляет пустую карту в игру, если ID не занят. ID, такой же, как и в остальных командах, и может быть от 0 до 255 (только обычно первые 10 карт заняты). Если ID занят, не делает ничего.
- `loadmap < ID > < ПУТЬ >` - Загружает карту с сервера на указанный ID.
- `rmmap < ID >` - Удаляет карту из раунда
---
- `pausemap < ID >` - Замораживает карту
- `unpausemap < ID >` - Размораживает карту
- `querymappaused < ID >` - Проверяет, заморожена ли карта или нет
---
- `tp < X > < Y > < ID >` - Телепортирует вас в координаты X,Y карты ID
- `tpgrid < ID > < X > < Y >` - Телепортирует грид с данным ID по текущей карте на координаты X, Y.
---
- `lsgrid` - Список гридов на текущей карте.
- `loadgrid < MAPID > < ПУТЬ > < X > < Y >` - Загружает грид на карту.
- `savegrid < ID > < ПУТЬ >` - Сохраняет грид на диск сервера. *На данный момент сломано.*
- `merge_grids < ID > < ID >` - Объединяет два грида в один.
- `showgridnodes` - Визуально показывает каждый грид.
---
- `forcemap < Map >` - Форсит следующий раунд (смену) на выбранной карте
