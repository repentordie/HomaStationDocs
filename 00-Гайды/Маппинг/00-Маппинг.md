## Терминология
**Маппинг** - то, чем мы ща будем заниматься, процесс создания карт для сс14.

**Мапа** - сама, неподсредственно, карта. В отличии от популярного мнения, в одном раунде (смене) присутсвует намного больше, чем одна мапа. Станция и всё что её окружает - это одна **Мапа**, ЦК - другая **Мапа**, Шахматное измерение - третья (да технически шахматное измерение скорее всего было подгружено в каждом раунде сски который вы играли в жизни). Игра переносит игроков между ними когда требуется, но помимо этого у **Мап** нет никакого интерфейса чтобы взаимодействовать *друг с другом*.

**Грид** - это подъединица **Мапы**. Если мы говорим о том, что игровой мир - **Мапа**, то Шаттл Карго, Эвак, Дереликты и, непосредственно, сама станция - это **Гриды**.
Каждый **Грид**  живёт отдельно от себя и может создаваться динамически  -- распилите станцию пополам, и она превратится в два **Грида**. Важное уточнение -- это работает в одну сторону. Когда ты пристыковываешь шаттл к станции, они не становятся одним **Гридом**, они просто двигаются вместе.

**Гриды** НЕ МОГУТ ПУТЕШЕСТВОВАТЬ МЕЖДУ МАПАМИ, ЭТО ХИТРЫЕ КОДИНГ ТРЮКИ.


## Комманды
### Маппинг
- mapping < ID > < ПУТЬ > - Запускает, собственно, процесс маппинга. Создаёт на указанном айди карту с диска сервера (если указан путь, иначе пустую), замораживает её чтобы можно было спокойно маппить и выдаёт призраку быструю панель строительства слева.

- savemap < ID > < ПУТЬ > - Сохраняет указанную карту по указанному пути на сервере.

Два важных момента.

1. Карту можно сохранять <h1>**ТОЛЬКО В ЗАМОРОЖЕННОМ СОСТОЯНИИ**</h1>
Если вы попытаетесь сначала протестировать и разморозить карту, а потом сохранить её - игра просто откажется это делать. Это хорошая практика по причине наличия в игре спавнеров и механики атмоса, электричества и всего тикающего со временем.

2. На данный момент, пока сервер в тестовой конфигурации, у вас есть **РЕАЛЬНЫЙ РИСК** потерять свою работу, если вовремя не сказать Лэпу перенести карту в обычные файлы. Сервер затирает всё, что не находится на Гите, а новые карты регистрируются в консоли только после перезапуска, что сотрёт их.

- mapinit < ID > - Размораживает карту для тестирования (по умолчанию, все карты созданные через mapping заморожены). Перед использованием, **сохраните карту на сервер**!

### Маппинг 2
- lsmap - Выдаёт список карт в раунде (смене)
- addmap < ID >  - Добавляет пустую карту в игру, если ID не занят. ID, такой же, как и в остальных командах, и может быть от 0 до 255 (только обычно первые 10 карт заняты). Если ID занят, не делает ничего.
- loadmap < ID > < ПУТЬ > - Загружает карту с сервера на указанный ID.
- rmmap < ID > - Удаляет карту из раунда

---

- pausemap < ID > - Замораживает карту
- unpausemap < ID > - Размораживает карту
- querymappaused < ID > - Проверяет, заморожена ли карта или нет
---
- tp < x > < y > < ID > - Телепортирует вас в координаты X,Y карты ID
- tpgrid < ID > < x > < y > - Телепортирует грид на текущей карте по координатам.
---
- lsgrid - Список гридов на текущей мапе.
- loadgrid < MAPID > < ПУТЬ > < X > < Y > - Загружает грид на мапу.
- savegrid < ID > < ПУТЬ > - Сохраняет грид на диск сервера. На данный момент сломано.
- merge_grids < ID > < ID > - Объединяет два грида в один.
- showgridnodes - Визуально показывает каждый грид.

### Мап-Смежное
- forcemap < Map > - Форсит следующий раунд (смену) на выбранной карте
